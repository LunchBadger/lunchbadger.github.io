Canvas

The LunchBadger platform Canvas is a single pane of glass where you can design, build and create your microservices and APIs.

The Canvas is broken up into a few easy parts to understand:

  • the entity palettte
  • quadrants
  • panels

Entity Palette

The left most toolbar is called the Entity Palette. Each icon in the palette represents a different entity that you can create. The entity palette is divided up into different entity types.

Some entities are used when you are developing your microservcies and integrating to existing data and application. Others are used when you want to expose microservices as APIs and secure them through an API gateway. More Details: Entity Palette

Quadrants

Quadrants represent different tiers within your application. They are designed to help you logically classify the different parts of your application in a microservices architecture.

Backend

The backend quadrant represents where you data and services exist and or persisted. Microservices can be built on top of existing legacy applications and or data.

There’s no need to throw away your old applications as you move into a microservices architecture. LunchBadger has tooling to help you create a new layer of microservices on top of your old applications. This will allow you to build entirely new microservices alongside microservices that sit as a layer on top of legacy.

LunchBadger utilizes the same tooling to connect to new data sources where you want to persist data utilized with new microserivces.

Example:

You have data in MySQL that exists as rows and tables and you want to create a set of microservices that represent that data. A MySQL connector can be created in the backend quadrant and represents a connection to MySQL as a legacy data source.

Example:

You are creating a new microservice that handles payments. The microservices requires each payment performed to be recorded with transaction details as a JSON object. A MongoDB connector can be created in the backend quadrant that represents the connection to new MongoDB instance that will store the new transaction data.

Private

The private quadrant represents all business functionality within your application. This quadrant contains all models, service endpoints and functions in your system. They are not exposed publically, but can communicate with each other. In LunchBadger communication with external services is done through the Gateway
